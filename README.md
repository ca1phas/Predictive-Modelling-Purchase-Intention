## 📦 Package Management (renv)

We use `renv` to ensure everyone is using the exact same package versions. This prevents "it works on my machine" errors.

1.  **Never run `renv::init()`.** The project is already initialized. Running `init()` will destroy the shared environment.
2.  **How to start working:** As mentioned above, simply running `scripts/00_main.R` handles the package restoration for you.
3.  **How to add a new package:** If you need a new package for your model (e.g., `install.packages("randomForest")`), you must update the lockfile so the rest of the team gets it:
    1.  Install your package: `install.packages("your_package")`
    2.  Test your code to make sure it works.
    3.  Take a snapshot: Run `renv::snapshot()` in the console (refer to the commented code at the top of `00_main.R`).
    4.  Commit the updated `renv.lock` file to your branch along with your script.

## 🛠️ Data Engineering Pipeline Summary (`01_data_engineering.r`)

### 1. Initial Cleaning & EDA

-   **125 duplicate rows** identified and removed to prevent data leakage.
-   Categorical columns cast to factors; `Month` ordered chronologically.
-   Confirmed **no missing values** across all 18 variables.
-   Distributions and skewness visualized and saved to `outputs/figures/`.

### 2. Train / Test Split

A **chronological time-based split** was applied to simulate real-world deployment where regular-season data predicts holiday-season purchasing behaviour:

| Split | Months    | Rows  | Share |
|-------|-----------|-------|-------|
| Train | Jan – Oct | 7,517 | 61.6% |
| Test  | Nov – Dec | 4,688 | 38.4% |

### 3. Feature Engineering & Cleaning

-   **VisitorType (Probabilistic Imputation):** The invalid `Other` class was replaced with a numeric feature `isReturningVisitor` (`1`=returning, `0`=new, `0.8749`=probabilistic value for `Other` based on train data).
-   **Missing Months (Quarter Binning):** `Month` was binned into Q1, Q2, Q3, and Q4 to prevent zero-variance dummy columns in LDA/Logistic Regression and handle unseen months in deployment.
-   **Rare Categorical Levels ("Other" Bucketing):** Levels in `OperatingSystems`, `Browser`, `Region`, and `TrafficType` with `< 5% frequency` were collapsed into an `"Other"` bucket to prevent singular matrix crashes.

### 4. Transformation & Encoding

-   **One-Hot Encoding:** Applied `fullRank=TRUE` to all nominal predictors to avoid the dummy variable trap.
-   **log1p Transformation:** Applied to all 10 numerical features to correct strong right skewness.
-   **Z-score Standardisation:** Fitted on training data only; learned means/SDs applied to both splits.
-   **Target Formatting:** `Revenue` remapped from `TRUE/FALSE` to `"Yes"/"No"` to ensure compatibility with `caret` internals.

### 5. Class Imbalance Handling

The target variable has an imbalance ratio of \~7:1 (12.4% purchase rate). Rather than a single global fix like SMOTE, model-specific strategies will be used:

| Model(s) | Strategy |
|----|----|
| Logistic Regression, Decision Tree | Class weights + threshold tuning |
| Naive Bayes, LDA | Balanced priors `c(0.5, 0.5)` + threshold tuning |
| Random Forest | `classwt` / `sampsize` + threshold tuning |
| XGBoost / LightGBM / CatBoost | Built-in balance parameters (`scale_pos_weight`, etc.) + PR-AUC metrics |

### 6. Preprocessing Artifacts Bundle (`preprocessing_artifacts.rds`)

Saved alongside the train/test datasets to ensure consistent, leakage-free transformation of new data in production. It contains: \* `prob_returning`: Probabilistic value for `VisitorType = "Other"` \* `production_keep_levels`: Rare-level bucketing rules per categorical column \* `oneh_model`: Fitted `dummyVars` OHE model \* `train_means` & `train_sds`: Feature means/SDs for standardisation

### Final Dataset Shape

| Split | Rows  | Features                 |
|-------|-------|--------------------------|
| Train | 7,517 | 32 predictors + 1 target |
| Test  | 4,688 | 32 predictors + 1 target |
